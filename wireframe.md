**개발 요청 기능**

앱 최초 실행 여부와 로그인 상태를 판단해 다음 화면으로 라우팅합니다.

- 최초 실행 → 서비스 소개(2번 화면)로 이동
- 재실행 (로그인 + 검사 완료) → 홈으로 이동
- 재실행 (비로그인) → 회원가입 화면으로 이동
- 재실행 (로그인 + 검사 미완료) → 무기력 검사로 이동
- 표시 시간: 약 2초 후 자동 전환
- 버전 표기 (현재 ver 2.0) 노출

## 5. 무기력 검사 (PHQ-9, 1/10 ~ 9/10)

> 9개 문항 모두 동일한 패턴입니다 (자가진단 1~9). 문항 텍스트만 다릅니다.

```
┌─────────────────────┐
│  9:41        ●●● 🔋 │
│  ←      질문 3/10   │
│  ████░░░░░░░░░░░░░  │
│                     │
│  잠들기가 어렵거나   │
│  자주 깼다 /         │
│  혹은 너무 많이 잤다.│
│                     │
│  ─────────────────  │
│                     │
│  ○ 전혀 그렇지 않다  │
│  ─────────────────  │
│  ○ 며칠 그랬다       │
│  ─────────────────  │
│  ● 자주 그랬다       │
│  ─────────────────  │
│  ○ 거의 매일 그랬다  │
│                     │
│  ┌────┐  ┌────────┐ │
│  │이전│  │  다음   │ │
│  └────┘  └────────┘ │
└─────────────────────┘
```

**개발 요청 기능 — PHQ-9 기반 무기력 진단**

PHQ-9 척도를 일상 언어로 순화한 9문항 설문으로 사용자의 무기력 상태를 측정합니다.

- 총 9문항 (자가진단 1~9)
- 응답: 4점 척도 (0~3점)
  - 0점: 전혀 그렇지 않다
  - 1점: 며칠 그랬다
  - 2점: 자주 그랬다
  - 3점: 거의 매일 그랬다
- 총점 범위: 0 ~ 27점
- 단계 구분
  - 없음: 0~4점
  - 낮음: 5~9점
  - 중간: 10~14점
  - 높음: 15점 이상
- 상단: 진행률 바 (현재 N/10) + 뒤로가기
- 하단 네비게이션: 이전 / 다음 (선택 시 다음 활성화)
- 답을 선택해야만 다음 버튼 활성화
- 9번 문항 완료 → 챌린지 선호 조사(6번)로 이동
- 문항 카드: 텍스트 길이에 따라 세로 자동 확장

**문항 (9개)**
1. 기분이 가라앉거나, 우울하거나, 희망이 없다고 느꼈다.
2. 평소 하던 일에 대한 흥미가 없어지거나 즐거움을 느끼지 못했다.
3. 잠들기가 어렵거나 자주 깼다 / 혹은 너무 많이 잤다.
4. 평소보다 식욕이 줄었다 / 혹은 평소보다 많이 먹었다.
5. 다른 사람들이 눈치 챌 정도로 평소보다 말과 행동이 느려졌다 / 혹은 너무 안절부절 못해서 가만히 앉아 있을 수 없었다.
6. 피곤하고 기운이 없었다.
7. 내가 잘못 했거나, 실패했다는 생각이 들었다 / 혹은 자신과 가족을 실망시켰다고 생각했다.
8. 신문을 읽거나 TV를 보는 것과 같은 일상적인 일에도 집중 할 수 없었다.
9. 차라리 죽는 것이 더 낫겠다고 생각했다 / 혹은 자해할 생각을 했다.

> 9번 문항(자살·자해 관련)은 점수가 1점 이상일 경우 결과 화면에서 전문 상담 안내(예: 1393 자살예방상담전화)를 함께 노출하는 것을 권장합니다.

> 오토레이아웃이 반영되어 있습니다.

**개발 요청 기능 — 챌린지 선호도 수집**

챌린지 추천 알고리즘에 사용할 사용자 선호도 5개 축을 **2지선다**로 수집합니다.

- 5개 축 × 양극단 2지선다 (단일 선택)
  - 활동: 정적인 / 역동적인
  - 관계: 혼자 / 함께
  - 방식: 성취중심 / 루틴중심
  - 취미: 창작 / 자기계발
  - 시간: 짧게 / 길게
- 각 축은 좌우 2개 박스(Pill)로 노출, 선택 시 `shallwe green` 활성 상태
- 5개 축 모두 응답해야 다음 버튼 활성화
- 다음 → 결과 분석 로딩(7번)
- 응답 데이터는 사용자 프로파일에 저장 → 추천 챌린지 매칭에 활용

> Figma 완성본 자가진단10은 양극 라벨 + 21x21 마커 2개 형식 — 본 문서의 명시적 Pill 2개와 시각적 의도가 동일 (선택 가능한 양극 중 하나). 구현 시 어느 시각으로 갈지 확정 필요.

**개발 요청 기능 — 진단 결과 + 첫 챌린지 진입**

PHQ-9 점수 → 4단계 분류 결과를 텍스트로 노출하고 다음 액션으로 유도합니다.

- 결과 텍스트 (동적 치환): "당신의 무기력 정도는 **{단계}** 입니다." (단계: 없음 / 낮음 / 중간 / 높음)
- 단계 구분 (PHQ-9 총점 기준): 없음 0~4 / 낮음 5~9 / 중간 10~14 / 높음 15+
- 단계별 메시지: 예 "ShallWe와 함께 조금씩 극복해나가요!" (단계별 카피 분기 가능)
- **CTA 2개 (세로 stacked, 위→아래)**
  - 위(Secondary): "다른 사람들의 챌린지를 구경하고 올래요" → 홈(9번, 모두의 챌린지 포커스)
  - 아래(Primary): "챌린지 시작" → 챌린지 탐색/추천 리스트(13번)
- 검사 결과는 사용자 프로필에 저장 (마이페이지에서 다시 조회 가능)

**개발 요청 기능 — 챌린지 인증 업로드**

오늘의 챌린지 완수를 사진 + 글로 인증하고 피드에 업로드합니다.

- 사진 입력: 카메라 직접 실행 또는 갤러리 선택 (1장 필수)
- **인증할 챌린지 선택** — 사진 영역 바로 아래 (Figma 완성본 일치)
  - 진행 중인 챌린지를 **chip 형식**(가로 wrap)으로 노출, 단일 선택
  - 1개면 자동 선택 + 비노출(접힘)
  - 2개 이상이면 강제 선택 (미선택 시 인증하기 비활성화)
  - chip이 화면 폭을 초과하면 자동으로 다음 줄로 wrap
  - 진행 중 챌린지가 5개 이상으로 많아지면 가로 스크롤 또는 ≡ 버튼으로 리스트 확장
  - 인증 진입이 홈의 "오늘의 챌린지" 카드에서 시작된 경우 해당 챌린지 사전 선택
- 글 작성: 텍스트 영역, 길이 제한 없음 (1000자 권장)
- 공개 설정
  - 공개: 모두의 챌린지 피드에 노출 + 마이페이지 내 게시글
  - 비공개: 본인만 (다이어리에는 표시)
  - 기본값: 공개
- 업로드 → 선택된 챌린지의 완수 일수 +1, 다이어리 캘린더에 표시, 홈으로 복귀
- 사진 미선택 또는 챌린지 미선택 시 업로드 버튼 비활성화
- CTA 라벨: **"업로드"** (Figma 완성본 일치). 진입은 캘린더·홈의 "인증하기" 버튼에서.

**개발 요청 기능 — 챌린지 추천 + 직접 만들기 진입**

선호도 조사 결과 기반 추천 챌린지를 **좌우 스와이프 캐러셀**로 노출합니다.

- 추천 카드 캐러셀
  - 가로 스와이프 (한 화면에 카드 1개, 양옆 살짝 보이기 옵션)
  - 카드 내용 (Figma 일치):
    - **짧은 타이틀** (예: "하늘 바라보기") — Sub Title 16
    - **액션 설명** (예: "하루에 한 번 하늘 보기") — Title 20 Bold
    - **미션** (예: "고개 들어서 하늘 한 번 보기") — Body 14
    - **타겟 디스크립션** (예: "스마트폰 보다 거북목 되기 직전인 당신에게") — Body 14
    - **기대효과** (예: "거북목 스트레칭과 한숨 돌리기") — Body 14
  - 하단 페이지 인디케이터 (●○○○○) — 추천 카드 개수만큼
  - 카드 탭 → 해당 챌린지 시작 (오늘의 챌린지로 등록 → 홈으로 이동)
- 캐러셀 아래 고정 위치에 **"챌린지 직접 만들기"** 진입 버튼
  - 탭 → 챌린지 만들기 1단계(14번)
- 추천 정렬: 사용자 선호도 + PHQ-9 단계 매칭 점수순
- 추천이 0개일 때: 캐러셀 자리에 "추천이 준비되지 않았어요" 빈 상태 + 직접 만들기만 노출

**개발 요청 기능 — 챌린지 진행 캘린더**

진행 중인 챌린지의 누적 완수 + 월별 캘린더를 한 화면에 보여줍니다.

- 상단 카드: 챌린지명, 진행률 (N일 / N00일), 진행 바, 최근 완수일 표시
- 캘린더: 월간 뷰, 셀(날짜)마다 **3가지 상태로 색상 구분**
  - **완수일**: `shallwe green` 채움 + 흰색 숫자
  - **미완수일** (이미 지난 날인데 인증 안 함): 흰색 배경 + 검정 숫자 + 회색 보더
  - **미래일** (아직 날짜가 지나지 않음): `Y Gray @ 20%` 회색 배경 + 회색 숫자 (탭 비활성)
  - 오늘 날짜는 보더 1.5px `shallwe green`으로 추가 강조 (위 3상태와 중첩 가능)
- 캘린더 하단 범례 (●완수 / ○미완수 / ░미래) 노출
- 날짜 탭
  - 완수일 → 게시물 상세(20번)로 이동
  - 미완수일 → "이 날은 인증 기록이 없어요" 토스트
  - 미래일 → 비활성 (반응 없음)
- 좌우 스와이프 또는 상단 화살표로 월 이동
- 월별 완수율 (예: 12/30) 부가 노출 가능
- 하단 "오늘 챌린지" 카드 (Figma 일치): 챌린지명 + **"인증하기" 버튼**
  - 인증하기 → 챌린지 인증 화면(12번)으로 이동

> Figma 완성본 캘린더는 셀당 색상 구분 없이 숫자만 표시 — 본 문서의 3상태(완수/미완수/미래) 색상 구분은 사용자 요청에 따른 추가. Figma 업데이트 필요.

## 글로벌 컴포넌트

### 하단 네비게이션 (Bottom Nav)

```
┌─────────────────────┐
│  📔   🏠    ✓    👤 │
│ 다이어리 홈  인증  마이│
└─────────────────────┘
```

- 4개 탭: 다이어리 / 홈 / 인증 / 마이
- 활성 탭은 메인 그린(`shallwe green`)으로 강조
- 회원가입·검사 등 온보딩 플로우, 챌린지 만들기, 검사 결과, 인증 업로드 등 풀스크린 플로우에서는 노출하지 않음

### 상단 헤더

```
┌─────────────────────┐
│  Shall We         + │
└─────────────────────┘
```

- 좌측: 서비스 로고 (Shall We 폰트)
- 우측: 챌린지 추가 버튼(+) — 홈 / 인증 / 마이 / 다이어리 탭에서 공통 노출
- 온보딩 / 모달 화면에서는 헤더 대신 뒤로가기(←)만 노출

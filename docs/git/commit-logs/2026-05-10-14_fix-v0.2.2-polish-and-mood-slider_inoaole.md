## 4. 변경 배경 및 결과

### 4.1 디자인 audit (11건)

`/plan-design-review` 실행 결과, 22 화면 + 22 컴포넌트 스캔 → 일관성 10건 + 폴리시 발견. 모두 적용 (사용자 "A" 선택).

**P1 일관성 (4건):**
- 온보딩 흐름의 bg drift: Splash gradient + Intro 1/2/3 white + Signup/Quiz/Result gray = 7-step에 3 종류 bg. Splash → bg-white로 unify (Intro와 첫 4 화면 통일).
- BackHeader sticky 불균일: 긴 스크롤 페이지 (PostDetail, Recommend, Quiz, Steps, Signup) 5개에 sticky 추가. 스크롤 후 ← 항상 접근 가능.
- 빈 상태 카피: Home + MyChallenge 두 곳이 같은 개념인데 동의어 사용 → "아직 시작한 챌린지가 없어요"로 통일. MyPage post empty는 다른 개념이라 보존.

**P2 폴리시 (3건):**
- EmptyState 차가운 "no items found" 안티패턴: Sparkles 기본 아이콘 + green-tint 원형 wrap + 패딩/타이포 정리.

- DiaryTab 최근 기록 mood 색상 누락: 캘린더는 색칠인데 리스트는 무색 → 3색 dot 추가로 시각 통일.

**P3 디테일 (1건):**

### 4.3 MoodSlider 신규

**Why:** 키워드 기반 mood inference는 사용자 의도와 안 맞을 수 있음 (예: 비꼬는 톤 / 키워드 회피). Figma 노드 `40000663:1050` 보니 슬라이더로 직접 mood 선택 → AI 분석 input.

**What:**
- `<input type="range" min=0 max=100>` 연속 드래그 (사용자 요청 — 처음 3-stop discrete였으나 풀어달라 요청)
- gradient track: `linear-gradient(red → gray → green)`
- 3 라벨 (부정적 / 보통 / 긍정적) — 현재 값이 속한 구간 강조
- 분류: `< 34` = negative, `34-66` = neutral, `> 66` = positive
- DiaryWrite 상단 카드에 통합 (질문 위)
- `moodFromScore(score)` 헬퍼로 저장 시 Mood 분류
- 기존 `inferMood(answers)` 호출 제거 (utils/diary.ts의 함수는 보존 — 향후 LLM fallback 가능)

**테스트 변경:**
- `DiaryWrite.test.tsx`의 negative 검증 케이스: 슬라이더 value를 0으로 설정하는 단계 추가
- `inferMood` 자체 테스트 (`tests/diary.test.ts`)는 그대로 (함수는 보존)

## 5. 중요 변경 사항

### MoodSlider 인터페이스

```ts
interface Props {
  value: number;             // 0-100
  onChange: (next: number) => void;
}

export function moodFromScore(score: number): Mood;
```

부모 (`DiaryWrite`)가 number 상태를 보유하고, 저장 시점에 `moodFromScore(score)`로 변환. 데이터 모델 (`DiaryEntry.mood: Mood`) 변경 없음.

### 라우팅 진입점 매트릭스

```
사용자 의도          → 진입점                     → 라우트
일기 쓰기            → bottom nav "AI 다이어리"    → /diary       (DiaryTab)
일기 작성 시작       → DiaryTab "오늘의 일기 쓰기" → /diary/write (DiaryWrite)
챌린지 진행 보기      → Home/My TodayCard           → /challenge   (MyChallenge)
인증 완료 후 확인     → ChallengeCert "인증" 완료    → /challenge   (MyChallenge)
인증 카메라         → /challenge "인증하기" 버튼     → /cert/upload (ChallengeCert)
```

### Settings 회원탈퇴 danger 처리

```tsx
<Row Icon={UserX} label="회원탈퇴" onClick={handleWithdraw} danger />
```

`Row` 컴포넌트에 `danger?: boolean` prop 추가 → 아이콘 + 텍스트 모두 `text-danger` (`#E5484D` Radix red 9). 위험 액션 시각 신호 강화.

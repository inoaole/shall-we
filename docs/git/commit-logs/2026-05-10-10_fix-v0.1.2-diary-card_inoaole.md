## 4. 변경 배경 및 결과

### 배경 (Why)

기존 다이어리 진행 카드는 챌린지 이름 + 진행률만 텍스트로 노출 → 이 화면이 "내가 시작한 챌린지"라는 정체성이 약함. Figma 참조 (node `40000611:5571` "7일 챌린지" 텍스트 포함 frame)는 로고 중심 카드 — 사용자가 다이어리 진입 시 "내 챌린지"가 시각적으로 brand되어 있음.

### 결과 (What)

진행 카드 구조:

```
┌────────────────────────────────┐
│         [logo 56×56]           │
│                                │
│      [pill: N일 챌린지]        │
│                                │
│         산책 챌린지             │
│                                │
│   N일 / 총일       완수 N일차   │
│   ▓▓▓▓░░░░░░░░░░░░░░          │
└────────────────────────────────┘
```

- 로고: `/logo/symbol.png` 56×56, 중앙 상단
- 뱃지: `bg-bg-green-tint text-primary` pill, durationDays 동적
- 제목: `text-title-20 text-ink`, 챌린지 이름
- 진행: 좌측 "N일 / 총일" + 우측 "완수 N일차" / "시작 전" (양 끝 정렬)
- progress bar: 카드 하단 full-width

스타일: `bg-white rounded-xl border border-gray/15 shadow-sm p-6` — design system Card 패턴 일치.

### 변경 안 한 것

- Calendar 섹션 (`이번 달 완수 현황`) 그대로
- 오늘 챌린지 + 인증하기 섹션 그대로
- Challenge 데이터 모델 (`icon` 필드 추가 안 함 — 앱 심볼을 모든 챌린지 공통 로고로 사용)
- 챌린지 이름 매핑 (mock의 shortTitle 그대로 사용)

## 5. 중요 변경 사항

### 진행/상태를 한 줄에 양 끝 배치

기존: 진행률(블록 1) + 상태("완수 - N일차") 라벨(블록 2) 세로 분리
변경: `flex justify-between items-baseline`로 한 줄. 좌측 숫자 강조 (primary 컬러), 우측 상태는 보조 텍스트(`text-body-12 text-gray`).

이유: 카드 세로 길이 줄임 + 두 정보 시각적 그룹화 (둘 다 "현재 진행 상태"라는 같은 카테고리).

### 로고 사용

Challenge 데이터에 icon 필드 추가하지 않음 — 앱 심볼 (`/logo/symbol.png`)을 모든 챌린지 공통 로고로 사용. 사용자 요청 "로고가 있어야 하고"가 챌린지별 아이콘이 아닌 앱 brand 노출로 해석. 향후 챌린지별 카테고리 아이콘 필요 시 데이터 모델 확장.

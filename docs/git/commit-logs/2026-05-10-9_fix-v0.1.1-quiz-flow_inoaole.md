## 4. 변경 배경 및 결과

### 배경 (Why)

배포 후 사용자 피드백 — 14 step 단일 흐름이 답답함 두 갈래:

1. **PHQ-9 9문항 매번 "다음" 탭** = 18번 탭 (선택 9 + 다음 9). 자가진단 특성상 답변이 직관적·빠른데 confirmation step이 흐름을 끊음.
2. **선호조사 5축이 1문항씩** = 흐름 분리 모호. "활동 / 관계 / 방식 / 취미 / 시간"은 사용자가 한눈에 비교하면서 자기 성향 그리는 게 자연스러움 (체크리스트 형태).

### 결과 (What)

**PHQ-9 phase (자동 진행):**
- 답 탭 → `bg-green-tint` 표시 250ms → 다음 step (`useRef` setTimeout)
- 하단 "다음" 버튼 제거. 작은 "이전" 버튼만 좌측 (`pair-prev` flex-1)
- 헤더 ← + 하단 "이전" 둘 다 작동 (오답 시 즉시 회복)
- 250ms는 Apple HIG selection-confirmation 권장 — 0ms는 twitchy, 500ms는 답답

**선호조사 phase (단일 화면):**
- 5축 카드 스택 (`bg-white rounded-xl border border-gray/15 p-5`)
- 각 카드: 카테고리 라벨 (활동/관계/방식/취미/시간) + PoleChoice
- 5개 모두 답해야 "결과 보기" 활성, 미답 시 disabled
- "이전" → PHQ-9 9번 문항으로 복귀 (답 유지)
- Progress bar/step 카운터 숨김 (단일 화면이라 진행률 개념 없음)

**총 흐름:** 14 step → 9 step + 1 화면 = 10 phase로 단축.

### 디자인 결정 (D1/D2/D3)

| 결정 | 채택 | 거절 |
|------|------|------|
| D1 자동 진행 타이밍 | 250ms + 선택 flash | 0ms (twitchy), 500ms (답답) |
| D2 PHQ-9 "이전" 위치 | 좌측 작게 (pair-prev) | 하단 full-width / 헤더만 |
| D3 선호조사 progress | 숨김 | "10/10" 표시 |

## 5. 중요 변경 사항

### setTimeout 누수 방지

```ts
const advanceTimer = useRef<ReturnType<typeof setTimeout> | null>(null);

useEffect(() => () => {
  if (advanceTimer.current) clearTimeout(advanceTimer.current);
}, []);
```

언마운트 + 빠른 뒤로가기 + 답 변경 시 모두 cleanup → 미진행 timer가 stale state로 화면 전환하는 버그 차단.

### 테스트 — fake timers

```ts
beforeEach(() => vi.useFakeTimers());
afterEach(() => vi.useRealTimers());

function flushAdvance() {
  act(() => { vi.advanceTimersByTime(260); });
}
```

자동 진행 검증을 위해 vitest fake timers 도입. 5 케이스 모두 이 패턴으로 통일.

### 테스트 셀렉터 — within(card)

각 선호 카드 내부에서만 radio 찾도록 `within(card).getAllByRole('radio')[0]` 사용. 초기 구현에서 `card.parentElement`로 한 단계 위 잡아 5축 전체 radio 잡혀 첫 번째만 갱신되는 버그 발견 → 수정.

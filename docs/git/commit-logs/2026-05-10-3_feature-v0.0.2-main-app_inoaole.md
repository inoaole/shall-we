## 4. 변경 배경 및 결과

### 배경 (Why)

- S1 (Foundation) + S2 (Onboarding) 완료 — 신규 사용자가 가입~결과 화면까지 진행 가능.
- S3는 22 화면 중 **나머지 12 화면**을 동시에 구현하는 가장 무거운 sprint (sprints/s3-main-app.md 32 task).
- 챌린지 라이프사이클(만들기 → 인증 → 캘린더 누적 → 마이페이지)이 동작해야 비로소 데모 가치 발생.
- 이전 사용자 피드백 1: "1393 안내는 빼줘" → SUICIDE_PREVENTION false 적용 (이번 cycle에 함께)
- 이전 사용자 결정: version 0.0.1 → 0.0.2 + 새 feature 브랜치

### 결과 (What)

신규 가입 + 검사 완료한 사용자가 다음 풀 cycle을 동작시킬 수 있음:

1. 결과 화면에서 "챌린지 시작" → 추천 가로 캐러셀 진입
2. 추천 카드 탭 → 즉시 ADD_CHALLENGE → 홈으로 복귀, 카드 노출
3. 또는 "직접 만들기" → 5단계 wizard → 완료 → 홈
4. 홈의 "오늘의 챌린지" 카드의 ✓ 버튼 → ChallengeCert
5. 사진 선택 → (1개일 땐 자동 / 2+개일 땐 chip 선택) → 글 작성 → 공개/비공개 → 업로드
6. CERTIFY dispatch → 다이어리 진입
7. 다이어리 캘린더의 오늘 셀이 초록 사각형으로 변함
8. 마이페이지의 "내 게시글"에 즉시 추가 노출
9. 설정 → 로그아웃/회원탈퇴로 RESET 가능

새로고침해도 sessionStorage v2 hydrate로 진행 상태 유지. 옛 v1 데이터(S2 형태)는 자동 무시되어 schema 충돌 없음.

## 5. 중요 변경 사항

### AppContext schema versioning (D-S2.1)

S2의 `STORAGE_KEY = 'shallwe.app.v1'` → S3 `'shallwe.app.v2'`. State에 challenges/posts/feed/prefs 추가되어 옛 v1 데이터로 hydrate 시 undefined 필드 → crash 위험. 키 이름만 바꾸면 옛 데이터는 자동 무시 (readStorage가 null 반환 → initialState 사용).

### PostFeed 공용 컴포넌트 (D-S3.4)

Home의 "모두의 챌린지" + MyPage의 "내 게시글"이 동일 visual + 다른 데이터 소스. `<PostFeed posts={...} viewMode={...} onToggleView={...} />`로 추출:
- `state.feed` (시드 + 본인 공개 posts) → Home
- `state.posts` (본인 모두) → MyPage

list/grid 뷰 토글 + 빈 상태 처리도 공용. 약 80 LOC 중복 제거.

### Recommend 카드 탭 (D-S3.2)

추천 카드는 `durationDays` 기본값을 데이터에 포함 (mocks/recommendations.json). 카드 탭 시 즉시 ADD_CHALLENGE → 홈으로 navigate. Steps wizard를 거치지 않으므로 추천은 추천답게, 커스터마이즈 원하면 "직접 만들기" 사용.

### ChallengeCert chip selector 0/1/2+ 분기 (D-S3.6)

진행 중 챌린지 개수에 따라 UI 분기:
- 0개: 진행 중 챌린지 섹션 자체 안 보임 + 업로드 버튼 비활성
- 1개: chip 미노출, "✓ 챌린지명" 텍스트로 자동 선택 표시
- 2+개: 모든 chip 노출, 강제 선택 필요 (선택 전엔 업로드 비활성)

이 핵심 로직을 `tests/ChallengeCert.test.tsx`로 회귀 보호. 첫 작성 시 `Seed` 컴포넌트가 render 중 dispatch해 무한 loop → useEffect로 dispatch 시점 변경 (test로 유발된 React anti-pattern 발견 + 수정).

### 사진 blob URL 안전 처리 (D-S3.1, P2)

ChallengeCert에서 `URL.createObjectURL(file)`로 blob URL 생성. unmount 시 `URL.revokeObjectURL`로 메모리 누수 차단 (useEffect cleanup). PostDetail에서는 새로고침 후 invalid blob URL이면 `onError` → ImageOff placeholder + "사진을 불러올 수 없어요" 메시지로 graceful fallback.

### crypto.randomUUID fallback (D-S3.7)

`utils/id.ts` 단일 진입점. `crypto.randomUUID` 지원 시 사용, 미지원 (Safari < 15.4 / Android Chrome < 92) 시 `id-{Date.now()}-{Math.random}` fallback.

### Steps wizard 단일 라우트 (D2)

`/create/new` 단일 URL + 내부 step state로 5단계. Back-button 안전 (step 1에서 navigate(-1)), deep link 단순. 새로고침 시 step 1로 reset (sessionStorage 임시 저장은 추후 polish).

### 1393 자살예방 안내 비활성화 (사용자 결정)

`data/result-messages.ts`의 `SUICIDE_PREVENTION.enabled = false`. 카드 자체는 코드에 남겨둠 (토글 가능). docs(wireframe.md / design.md)에 권장 표기는 그대로 — 추후 결정 변경 시 enabled만 true로 바꾸면 됨.

## 7. 요약

ShallWe React 프로토타입의 핵심 기능이 모두 동작합니다. 이번 sprint 작업으로 사용자가 다음 일상 cycle을 처음부터 끝까지 경험할 수 있게 되었습니다:

1. **챌린지 만들기** — 추천 5개 중 골라서 시작하거나, 5단계 wizard로 직접 만들기
2. **인증** — 사진 찍고 / 진행 중 챌린지 선택(여러 개면 골라서) / 글 쓰고 / 공개·비공개 결정 → 업로드
3. **다이어리에서 누적 확인** — 캘린더에 오늘 셀이 초록 사각형으로 표시, 진행률 바 갱신, 며칠 완수했는지 한눈에
4. **마이페이지에서 내 게시글 관리** — 본인이 쓴 모든 게시글(공개+비공개) 리스트로 확인, 리스트/그리드 뷰 전환
5. **설정** — 알림 / 약관 / 로그아웃·회원탈퇴 메뉴

새로고침해도 진행 상태가 유지되고, 옛 데이터로 화면이 깨지는 일이 없도록 데이터 형식 버전을 v2로 올렸습니다. 사진은 새로고침하면 보안상 사라지지만, 이때 빈 화면 대신 "사진을 불러올 수 없어요" 안내가 나옵니다.

이전 cycle에서 결정하신 1393 자살예방 안내는 이번 commit으로 비활성화되었고, 코드 토글로 남겨두어 나중에 결정이 바뀌면 한 줄만 바꾸면 됩니다.

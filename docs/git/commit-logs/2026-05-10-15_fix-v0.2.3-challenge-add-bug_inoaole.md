## 4. 변경 배경 및 결과

### 배경 (Why)

**버그 시나리오:**
1. 사용자가 챌린지 A 추가 → `state.challenges = [A]` → Home에 A 표시 ✓
2. 사용자가 챌린지 B 추가 → `state.challenges = [A, B]` (append) → Home은 `state.challenges[0]` = **여전히 A** 표시 ✗
3. 사용자: "방금 추가한 거 어디 갔지?"

**근본 원인:** 리듀서가 append (`[...state.challenges, action.payload]`)인데 모든 소비자가 `[0]`만 읽음.

```bash
$ grep -rn "state.challenges\[0\]" src
src/screens/home/Home.tsx:12
src/screens/my/MyPage.tsx:12
src/screens/challenge/MyChallenge.tsx:25
```

**부수 발견 (mockup 정합):** 사용자가 이전에 보낸 Home mockup ("산책 10분 하기" + "하늘 바라보기" 2개 동시 노출). 코드는 1개만 보여줌 → 디자인 의도와 불일치.

### 결과 (What)

**Fix 1 — 리듀서 prepend:**
```ts
- return { ...state, challenges: [...state.challenges, action.payload] };
+ return { ...state, challenges: [action.payload, ...state.challenges] };
```

소비자 코드 변경 없이 `[0]`이 newest. MyChallenge가 자동으로 새 챌린지로 전환됨 (single-focus라 자연).

**Fix 2 — Home/MyPage 다중 노출:**
```tsx
{state.challenges.length > 0 ? (
  <section>
    <h2>오늘의 챌린지</h2>
    <div className="space-y-3">
      {state.challenges.map((c) => (
        <TodayCard key={c.id} ... onClick={() => navigate('/challenge')} />
      ))}
    </div>
  </section>
) : <EmptyState .../>}
```

각 카드는 자기 `completedDaysFor(c.id)` 계산. 모든 카드 같은 진입점 (`/challenge`)으로 navigate — MyChallenge가 `[0]` (newest)을 표시하므로 사용자가 진행하던 챌린지 우선.

**MyChallenge는 그대로:** `state.challenges[0]` 유지. 단일 focus 화면이라 모두 표시 안 함. prepend 덕에 자동으로 가장 최근 챌린지 노출.

## 5. 중요 변경 사항

### Reducer 한 줄 + 주석

```ts
case 'ADD_CHALLENGE':
  // Prepend so [0] is the newest. Home/MyPage/MyChallenge consumers
  // read newest-first; old append behavior buried new challenges.
  return { ...state, challenges: [action.payload, ...state.challenges] };
```

근본 원인 + WHY를 코멘트로 남겨 향후 재발 방지.

### Home.test.tsx — 회귀 가드

`Seed` 컴포넌트로 useEffect에서 ADD_CHALLENGE dispatch → Home 렌더 → DOM 검증.

3 케이스:
1. 빈 상태 → EmptyState
2. 1개 → TodayCard 1개
3. 2개 → TodayCard 2개 + newest first 순서 검증

DOM 순서 검증 코드:
```ts
const cards = screen.getAllByRole('button').filter((b) =>
  b.textContent?.includes('일 챌린지'),
);
expect(cards[0].textContent).toContain('산책 10분');  // 마지막 추가
expect(cards[1].textContent).toContain('하늘 보기');  // 먼저 추가
```

리듀서를 다시 append로 돌리면 이 테스트가 깨짐 → 영구 회귀 가드.

## 4. 변경 배경 및 결과

### 4.1 TodayCard (홈 + 마이)

**Why:** 사용자가 챌린지 진행률을 텍스트만으로 인지하기 어려움. Figma는 progress bar + 우측 brand mark로 한눈에 진행 + 정체성 표현.

**What:**
- Progress bar 추가 (`completedDays / durationDays`)
- 우측 40×40 앱 로고
- 인증하기 CheckCircle 제거 → bottom nav 인증 탭으로 일원화
- rightSlot prop 제거 (사용처 없음)

### 4.2 다이어리 mood 캘린더

**Why:** 사용자 피드백 — "기존에 있던 챌린지 눌렀을 때 보이는 캘린더랑 동일하게 만들어줘, 지금 스타일이 달라서". 두 화면이 같은 캘린더 컴포넌트를 쓰는데 셀 스타일이 다르면 학습 비용 증가.

**What:** Mood 셀을 챌린지 셀과 동일한 `w-10 h-10 rounded-md` 사각형으로 통일. 색상 매핑:
- positive → `bg-primary text-white font-semibold` (challenge done과 동일)
- neutral → `bg-bg-green-tint text-ink`
- negative → `bg-yellow text-ink`
- empty → `bg-white border border-gray/30 text-ink`
- future → `bg-bg-gray text-gray/60`

이전 시도 (`rounded-2xl h-12` + 셀 안 로고)는 챌린지 화면과 시각 분리를 너무 강하게 만들어 오히려 일관성 해침 → 폐기.

### 4.3 추천 카드 + 확인 다이얼로그

**Why:** Figma 노드 `40000625:206` 이름이 "챌린지 추가 - **팝업**" — 추천 카드 탭하면 확인 모달이 떠야 함. 또한 사용자가 스크린샷으로 카드 자체도 옐로 hero + 아이콘 + 미션 pill 형태로 변경 요청.

**What:**

**추천 카드 (carousel item):**

│  ┌──────────────────┐  │

│  └──────────────────┘  │
│  하루에 한 번 하늘 보기  │  ← 타이틀
│  [미션 : ... 회색 pill] │

## 5. 중요 변경 사항

### Modal 컴포넌트

```tsx
useEffect(() => {
  if (!open) return;
  const handler = (e: KeyboardEvent) => e.key === 'Escape' && onClose();
  window.addEventListener('keydown', handler);
  document.body.style.overflow = 'hidden';
  return () => {
    window.removeEventListener('keydown', handler);
    document.body.style.overflow = prevOverflow;
  };
}, [open, onClose]);
```

- 열림 상태에서만 keydown / overflow 잠금 등록 → unmount + open=false 변경 시 cleanup
- `prevOverflow` 보존으로 다른 컴포넌트가 overflow 설정한 경우 복원
- `role="dialog"` + `aria-modal="true"` + `aria-label`

### RecommendCard 인터페이스 변경

기존: `shortTitle / action / mission / target / effect`
변경: `action / mission / target / effect / durationDays / Icon`

`shortTitle`은 다이얼로그 헤더에서만 노출 (카드에는 action만). Icon은 컴포넌트 props로 받아서 부모(Recommend)가 `recIcon(rec.icon)`로 resolve.

dev/Components.tsx의 showcase RecommendCard도 새 시그니처 적용 (Cloud icon).

### Home의 인증하기 버튼 제거

기존: TodayCard rightSlot에 CheckCircle 28px → 클릭 시 `/cert/upload`. 사용자 피드백으로 brand 로고가 그 자리를 차지해야 함 → CheckCircle 제거.

영향: 홈에서 한 탭으로 바로 인증 진입 못함. 대안: 하단 탭 인증 또는 `/challenge` → 인증하기 버튼. 사용자가 명시적으로 받아들임.

## 4. 변경 배경 및 결과

### 배경 (Why)

- S1에서 React 뼈대 + 12 UI primitive + 17 placeholder 화면 + 라우팅 동작.
- 이제 22 화면 중 첫 8개(온보딩)를 실제로 동작시킴.
- 신규 사용자가 처음 들어와서 → 가입 → 무기력 검사 → 결과까지 끊김 없이 클릭만으로 완주하게 만드는 것이 S2 목표.
- PHQ-9 9문항 + 챌린지 선호조사 5축 = 14 step을 어떻게 구현할지가 핵심 설계 결정 (eng review에서 D1+D2 합의).

### 결과 (What)

- `/` (Splash) → `/onboarding/intro/1` → `/onboarding/intro/2` → `/signup` 또는 `/check/quiz` → 14 step → `/check/analyzing` → `/check/result` 풀 진행.
- PHQ-9 점수 → 4단계 (없음/낮음/중간/높음) 정확히 분기, 단계별 색상·메시지·1393 안내 분기 동작.
- 새로고침해도 sessionStorage hydrate로 진행 상태 유지 (Splash 라우팅이 적절히 분기).
- back-button으로 Quiz 도중 돌아가도 답이 유지됨, Result에서 back 시 Analyzing으로 돌아가지 않음 (replace 적용).
- 자동 테스트 9개 추가 (5 단위 + 4 RTL 통합).

### 시각 polish (사용자 요청 "퀄리티 올려줘"에 대한 응답)

S2 첫 구현 후 시각 검증 → quality 부족 부분 즉시 수정:
- **Button disabled**: 회색 위 흰 텍스트는 거의 안 보임 → text-gray로 변경
- **Splash/Intro1/Intro2**: 평면 흰 배경 → bg-gray → bg-green-tint 그라디언트 + symbol을 white-circle + shadow에 wrap → atmosphere 형성
- **Quiz**: 단계 indicator가 약함 → phase label("무기력 검사" / "챌린지 선호 조사") + progress bar 두께 강화 + white frame 안에 배치
- **Result**: "낮음" 작은 pill + "입니다" 분리 어색 → 40px 큰 단계명 + ring + shadow + white card에 메시지

## 5. 중요 변경 사항

### Quiz 14 step 단일 컴포넌트 (D1+D2)

기존 plan에서는 PHQ-9 9 화면 + 선호조사 1 화면(5축 한 번에) = 10 화면이었으나, eng review D1에서 **데이터 주도 14 step 단일 컴포넌트**로 통합. 코드량 70% 감소 + 신규 문항 추가가 데이터만 수정하는 일이 됨.

D2의 단일 라우트 + 내부 step state 적용으로:
- URL은 `/check/quiz` 하나
- back button으로 step 1까지 돌아가면 navigate(-1)로 직전 화면(Intro2 또는 Signup)으로 빠짐
- Deep link `/check/quiz` 진입 시 항상 step 1부터 (잘못된 deep link로 뒷 step 진입 가능성 차단)

### sessionStorage 자동 persist (D-S2.1)

`AppContext`의 useEffect로 매 dispatch마다 sessionStorage 자동 저장. STORAGE_KEY에 버전 suffix(`v1`) 부여 — S3에서 challenges/posts 추가 시 v2로 bump하면 옛 데이터 깨끗하게 무시. quota 초과 시 console.warn (S4에서 토스트 통합).

### back loop 차단

Quiz → Analyzing → Result 흐름에서 Result에서 back 시 Analyzing으로 돌아가면 다시 timer가 돌아 무한 루프 발생. 두 navigate 모두 `{ replace: true }` 적용해 history entry를 교체 → back 시 Result에서 Splash 또는 Intro로 빠짐.

### Mock OAuth 처리 (§8-3)

Intro2와 Signup의 "구글로 계속" 버튼 → 백엔드 없으므로 즉시 dispatch SIGNUP({nickname: '데모유저', email: 'demo@shallwe.app'}) + console "demo mode" 메시지 + Quiz로 navigate. UT 시 OAuth 흐름 없이도 다음 단계 진입 가능.

### 1393 자살예방 안내 (§8-4)

PHQ-9 9번 문항 응답이 1점(며칠 그랬다) 이상일 때 Result 화면에 자살예방상담전화 1393 카드 노출. `tel:` 링크 + 24시간 무료 상담 안내. `data/result-messages.ts`의 `SUICIDE_PREVENTION.enabled` 플래그로 제어 가능.

## 7. 요약

ShallWe React 프로토타입의 첫 사용자 플로우가 동작합니다. 2번째 sprint 작업으로, 신규 사용자가 앱을 처음 열면 다음 흐름을 끊김 없이 경험합니다:

1. **앱 첫 실행** — Shall We 로고가 떠 있는 화면 2초 후 자동 진입
2. **서비스 소개** — '쉬었음 청년을 위한 AI 감정기록 챌린지 서비스' 한 줄 소개
3. **검사 안내** — 가입 또는 구글 로그인 선택 (구글은 데모 모드로 자동 가입)
4. **회원가입** — 닉네임·이메일·비밀번호 입력 (실시간 검증)
5. **무기력 검사** — PHQ-9 9문항 + 챌린지 선호 5문항 = 14단계 (이전/다음 자유롭게 이동, 답 안 하면 다음 비활성)
6. **결과 분석** — "당신의 마음을 읽고 있어요" 메시지 1.5초 + 점수 계산
7. **결과** — 4단계 중 본인 단계가 큰 색상 pill로 노출 + 단계별 격려 메시지 + 챌린지 시작 vs 다른 사람 구경하기 선택

특별히 9번 문항(자해 관련)에 응답이 있으면 1393 자살예방상담전화 안내 카드가 함께 노출됩니다.

화면 디자인은 첫 구현 후 사용자 피드백 ("퀄리티 올려줘")을 받아 즉시 1라운드 다듬기 완료 — 그라데이션 배경 / 로고를 흰 원에 wrap / 단계 표시 강화 / 결과 화면의 단계명을 시선이 가는 큰 pill로 강조했습니다.

새로고침해도 진행 상태가 유지되어 어디서 중단했든 다시 그 위치로 자동 복귀합니다.

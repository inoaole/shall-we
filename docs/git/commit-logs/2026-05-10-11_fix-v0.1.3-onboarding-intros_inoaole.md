## 4. 변경 배경 및 결과

### 배경 (Why)

Figma 노드 `40000603:1690` ("온보딩_2") 확인 — 인트로는 흰 배경 + 중앙 텍스트 + 3-dot pagination의 미니멀 carousel 패턴.

기존 구현 비교:
- **Intro1**: 로고 + 그라데이션 + 텍스트 + 다음 = 시각 요소 과다
- **Intro2**: "먼저 무기력 검사를 진행해요" 헤드라인이 가입/구글 버튼에 묻힘 = 인트로의 정체성 약함

사용자 피드백: "이거 두개 화면 온보딩에서 없어서 추가해줘" — 즉 Figma 스타일 깨끗한 인트로 화면이 없다는 의미.

### 결과 (What)

3-step carousel로 분리:

| 화면 | 텍스트 | DotPagination | CTA |
|------|--------|---------------|-----|
| Intro1 | "'쉬었음 청년'을 위한 / AI 감정기록 챌린지 서비스" | ●○○ | 다음 |
| Intro2 | "먼저 무기력 검사를 / 진행해요" | ○●○ | 다음 |
| Intro3 | 로고 + "지금 시작해볼까요?" | ○○● | 가입하기 / 구글로 계속 |

**디자인 원칙:**
- 흰 배경 (`bg-white`) — Figma 정합
- 텍스트만 중앙 정렬, `text-title-24 text-ink leading-relaxed`
- DotPagination: active 점은 `w-5` pill (가로로 늘어남), idle은 `w-2` dot — 진행도 + 단계 모두 시각화
- Intro3에만 로고 노출 (가입 CTA 화면에서 brand 마지막 강조)

### 변경 안 한 것

- Splash 라우팅 로직 (`!nickname → intro/1`) — 그대로
- Signup 페이지 (`/signup`) — 그대로 (Intro3에서 진입)
- Quiz / Analyzing / Result — 그대로

## 5. 중요 변경 사항

### 가입 CTA 분리

기존 Intro2 하단의 "가입하기 / 구글로 계속" 묶음 + `handleGoogle` 디스패치를 Intro3로 옮김. 데모 모드 동작 (`SIGNUP` dispatch + `notify.demoMode` + `/check/quiz`로 redirect) 그대로 유지.

### DotPagination 컴포넌트

3 화면 공유. `current` 1-based, `tablist` role + `aria-selected` 접근성 라벨링. `transition-all`로 활성 점이 늘어나는 애니메이션 (200ms 기본).

### Splash → Intro1 라우팅

Splash가 nickname 없을 때 `/onboarding/intro/1`로 보내므로 새 흐름 자동 적용. 기존 사용자(nickname 있음)는 영향 없음.

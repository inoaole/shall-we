## 4. 변경 배경 및 결과

### 배경 (Why)

직전 polish 라운드 (S1·S2 polish)에서 atmosphere · 시각 강조를 추가하는 방향으로 다듬었음. 결과적으로 일부 화면이 Figma 완성본의 절제된 미감보다 무거워졌고, design.md spec에 없는 장식 요소(yellow 추천 배지 등)가 추가됨.

사용자 피드백 "퀄리티 figma 기반으로, design.md 기반으로 더 올려줄래"는 visual richness가 아닌 confidence (절제) 방향의 polish를 요청.

### 결과 (What)

7개 over-decorated 요소 제거 → Figma 클린 미감 회복:

- **Splash**: 떠다니는 그린/노랑 blob 2개 제거. 그라디언트만으로 충분한 atmosphere.
- **Result**: 40px green pill + ring + shadow 강조 제거. Figma 완성본처럼 "당신의 무기력 정도는 / 낮음 입니다" 텍스트 hierarchy로 환원, '낮음'만 그린 컬러로 강조.
- **Quiz**: 헤더의 "무기력 검사" / "챌린지 선호 조사" phase label 제거. Figma는 "← 질문 N/10"만 단순 노출.
- **TodayCard**: 좌측 그린 accent strip + Sparkles 아이콘 pill 배지 제거. Figma 일치하는 깨끗한 단일 카드 + "벌써 N일 완수!" 텍스트.
- **RecommendCard**: 우상단 노란 "추천" 배지 제거 (Figma 완성본 미존재). 미션·대상·기대효과를 dl/dt/dd 정돈된 그리드로 변경.
- **AddCard**: 점선 dashed border + 원형 아이콘 wrap 제거. bg-green-tint 솔리드 + 단순 + 아이콘.
- **BottomNav**: active 탭에 icon fill + font-semibold 추가 (단순 색상 변경 외에 명확한 시각 신호).

### 일관 통일 (Surface treatment)

기존 카드들이 shadow-sm / shadow-md / dashed border 혼용. 이번 라운드로 모든 카드가 `bg-white + border border-gray/15 + shadow-sm`로 통일 (RecommendCard / FeedCard / TodayCard / Calendar 컨테이너 동일). AddCard만 의도적으로 bg-green-tint로 시선 분기.

## 5. 중요 변경 사항

### Subtraction default (Rams 원칙) 적용

> "As little design as possible. Less, but better."

이전 polish는 시각 강조 추가에 집중 (그린 accent, 노란 배지, ring shadow). 이번은 반대 방향 — 콘텐츠가 말하게 하고 장식은 빼기. 결과적으로 Figma 완성본 (절제된 텍스트 중심)과 톤 일치.

### Figma 완성본 vs design.md spec divergence 정리

design.md §5.3에 "RecommendCard 우상단 추천 배지 (yellow)"라고 명시했으나 Figma 완성본 챌린지추가 frame에는 해당 배지 미존재. design.md가 임의 spec이었음을 인정하고 Figma 우선으로 정렬. (design.md는 후속 commit에서 정리 검토).

### Card surface 일관성

이전: `shadow-md` 강한 그림자 + 다양한 padding.
이후: `border border-gray/15 + shadow-sm` 통일 + 모든 일반 카드 padding 16pt (design.md §5.3 spec).
효과: 화면 내 카드 위계가 더 명확해짐 (TodayCard만 약간 강조하고 싶을 땐 padding으로 조절).

### BottomNav active state 강화

이전: 활성 탭이 색상만 변함.
이후: 색상 + icon fill (15% opacity) + font-semibold.
근거: design.md §5.6에 active 탭은 "primary 컬러" 명시. 단순 색상 변경만으로는 약한 신호 → fill로 시각 weight 추가.

## 7. 요약

ShallWe 프로토타입의 시각 디자인을 한 단계 더 정돈했습니다. 이번 작업의 방향은 "더 화려하게"가 아니라 "더 절제되게" — 화면을 Figma 원본의 깔끔한 미감에 더 가깝게 환원했습니다.

구체적으로:

- 결과 화면의 "낮음" 강조를 큰 색상 pill에서 단순 텍스트(녹색 한 단어)로 줄여 Figma 디자인과 동일하게 만들었습니다.
- 검사 화면 위 "무기력 검사 / 챌린지 선호 조사" 머리말을 제거해 질문에만 집중할 수 있게 했습니다.
- 챌린지 카드의 좌측 녹색 줄, 추천 카드의 노란 "추천" 배지 등 장식 요소를 정리해서 모든 카드가 같은 스타일을 공유합니다.
- 시작 화면의 떠다니는 색깔 원 두 개를 빼고 그라데이션만 남겼습니다.
- 하단 4탭의 활성 표시를 더 또렷하게 (아이콘이 살짝 채워지고 글자가 굵어짐).

전체적으로 화면이 차분해지고 콘텐츠가 더 잘 읽히게 됐습니다. 코드량도 6줄 줄었고 (장식 코드 정리), 테스트는 그대로 19/19 통과합니다.

## 4. 변경 배경 및 결과

### 배경 (Why)

기존 `다이어리` 탭은 챌린지 완수 캘린더 + 진행 카드 = "내 챌린지 진행 현황" 화면. 사용자가 매일 감정 기록을 위해 진입할 동기가 약함.

Figma 신규 디자인은 다이어리 탭을 **AI 감정 일기** 도구로 재정의:
- 매일 3 질문에 답하면 mock LLM이 감정 분석 + 응원 + 추천 챌린지 제안
- 캘린더는 "이번 달 어떤 감정으로 보냈는지" mood marker 표시
- 챌린지 진행 현황은 별도 화면 (`/challenge`)으로 분리, 홈의 TodayCard에서 진입

### 결과 (What)

**라우팅:**
```
Tab /diary  → DiaryTab (mood 캘린더 + 일기 쓰기 CTA + 최근 기록)
/diary/write → DiaryWrite (write → analyzing → result, 단일 컴포넌트)
/challenge   → MyChallenge (옛 다이어리 컨텐츠 그대로)
```

**작성 흐름 (Quiz와 동일 패턴 D1):**
```
write     → 3 textarea + "AI 감정 분석 보기" (모두 답해야 활성)
analyzing → "마음을 읽고 있어요" 1.5s → "따뜻한 피드백을 준비할게요" 1.5s
result    → 감정 라벨 + 응원 메시지 + 도움됨/공감안됨 + 추천 챌린지 + 홈으로
```

`useRef` setTimeout cleanup으로 unmount/뒤로가기 시 race 차단.

**Mood 추론 (D5 keyword-based):**
- 부정 키워드 (힘들/무기력/우울/지쳐 등 22개) vs 긍정 키워드 (감사/좋/행복/기쁨 등 22개) 카운트 비교
- 동수 또는 빈 답 → neutral
- 향후 LLM 연동 시 `src/utils/ai.ts`로 분리 예정 (현재는 fallback 데이터로 보존)

**Mood 캘린더 셀 (Figma 정합):**
- 긍정 → `bg-bg-green-tint` + 날짜 아래 작은 앱 로고 (`/logo/symbol.png` 14×14)
- 중립 → `bg-gray/10`
- 부정 → `bg-yellow/40`
- 빈 셀 → `bg-bg-gray`
- 셀 모양 `rounded-2xl`, 높이 `h-12` (모든 셀 동일, 그리드 균일)
- 오늘 → `ring-primary` 강조

**Storage 마이그레이션 (D2):**
v2 데이터 발견 시 → `{ diaries: [], ...v2parsed }` 형태로 v3 키에 저장 + v2 키 삭제. 기존 사용자 챌린지/포스트/온보딩 상태 보존.

**Calendar 컴포넌트 generic화 (D4 DRY):**
```ts
interface CalendarProps<C extends { day: number; isToday?: boolean; date: Date }> {
  cells: C[];
  renderCell: (cell: C, onClick: () => void) => ReactNode;
  ...
}
```
challenge는 `<CalendarCell state={c.state} />` 기본 사용, mood는 커스텀 button 렌더. 그리드 + 헤더 로직 공유.

### 변경 안 한 것 (NOT in scope)

- 실제 LLM 연동 — keyword-based mock 사용
- 다이어리 entry 상세 보기 화면 (`/diary/:id`)
- 다이어리 entry 삭제/수정
- "공감이 안 돼요" → AI 메시지 재생성 (현재는 helpful field에 false 저장만)
- 추천 챌린지 화면을 mood로 필터링 (현재 `/create`는 PHQ-9 기반)

## 5. 중요 변경 사항

### 데이터 모델 확장

```ts
type Mood = 'positive' | 'neutral' | 'negative';

type DiaryEntry = {
  id: string;
  date: string;
  answers: { q1: string; q2: string; q3: string };
  mood: Mood;
  aiFeedback: string;
  recommendedChallenge: string;
  helpful: boolean | null;
};

// State에 diaries: DiaryEntry[]
// Action: ADD_DIARY, RATE_DIARY
```

### Storage v2 → v3 마이그레이션

```ts
const PREV_KEYS = ['shallwe.app.v2'];

readStorage 시:
1. v3 키 hit → 그대로 반환
2. miss → PREV_KEYS 순회, 첫 hit을 { diaries: [], ...prev }로 v3에 복사 + 옛 키 삭제
3. 모두 miss → null (clean start)
```

향후 v3 → v4 시 PREV_KEYS에 v2 + v3 추가, 추가 필드 default만 명시하면 됨.

### setTimeout cleanup

```ts
const stage1Timer = useRef<ReturnType<typeof setTimeout> | null>(null);
const stage2Timer = useRef<ReturnType<typeof setTimeout> | null>(null);

useEffect(() => () => {
  if (stage1Timer.current) clearTimeout(stage1Timer.current);
  if (stage2Timer.current) clearTimeout(stage2Timer.current);
}, []);
```

DiaryWrite analyzing 중 뒤로가기 → 모든 timer cleanup → result phase 도달 안 함.

### Home 중복 버튼 제거

초기 구현 시 "내 챌린지 진행 보기" 별도 버튼 추가 → 사용자 피드백 ("TodayCard 자체가 이미 같은 곳으로 가서 반복적") → 제거. TodayCard onClick → `/challenge` 단일 진입점.

## 7. 요약

다이어리 탭의 정체성을 "챌린지 진행 현황"에서 **"AI 감정 일기 도구"**로 재정의. 매일 3 질문 → mock LLM 분석 → 응원 + 추천 챌린지 흐름을 추가하고, 챌린지 진행 화면은 홈에서 진입하는 별도 라우트로 분리. 캘린더 컴포넌트를 generic 화해 두 화면이 같은 그리드 로직 공유. Storage v2→v3 마이그레이션으로 기존 사용자 데이터 보존. v0.2.0 minor.

## 4. 변경 배경 및 결과

### 배경 (Why)

Vercel "New Project" UI에서 GitHub 레포 import 직전 사용자가 "Vite preset이 맞아?" 확인 → preset OK 확인 + SPA fallback 누락 위험 안내.

React Router v6 사용 중이라 `/home`, `/diary` 등 deep link 직접 접속 시 (또는 새로고침 시) Vercel 기본 정적 호스팅이 404 반환. `vercel.json` 없이 배포하면 Splash 빼곤 모든 페이지에서 새로고침 깨짐.

또한 0.0.x 누적이 7 PR (S1+S2+S3+polish×2+mypage+cleanup) — 이제 안정화 단계라 minor 0.1.0 bump가 자연스러운 시점.

### 결과 (What)

`vercel.json`이 다음 보장:
1. **SPA fallback** — 모든 경로(`/(.*)`)를 index.html로 rewrite → React Router가 클라이언트에서 처리
2. **정적 자원 캐싱**
   - `/assets/*` (Vite hash 포함 JS/CSS) → 1년 immutable (안전, hash 바뀌면 새 파일)
   - `/logo/*` (PNG) → 1일 (디자인 변경 시 빠르게 반영)
3. **framework 명시** — Vercel auto-detect와 동일하지만 explicit 안전

이제 `git push` 후 Vercel "Deploy" 누르면 5분 안에 production URL 받음. 배포 후:
- 모든 22 화면 deep link 정상 (`/home`, `/check/quiz`, `/post/abc` 등)
- 새로고침 시 404 없음
- Lighthouse 성능 점수 ↑ (캐시 헤더 효과)

## 5. 중요 변경 사항

### vercel.json schema

```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "framework": "vite",
  "rewrites": [{ "source": "/(.*)", "destination": "/" }],
  "headers": [
    { "source": "/assets/(.*)", "headers": [
      { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
    ]},
    { "source": "/logo/(.*)", "headers": [
      { "key": "Cache-Control", "value": "public, max-age=86400" }
    ]}
  ]
}
```

`framework: "vite"`는 Vercel이 자동 감지하지만 explicit 명시가 안전. rewrite는 single-page app의 표준 패턴 — 모든 경로를 `/` (index.html)로 보내고 클라이언트 라우터가 화면 결정.

### 0.1.0 minor bump 의미

semver 관점에서 0.x.y는 "pre-1.0 unstable"이지만 우리 컨벤션상:
- 0.0.x: 개발 중 패치 (각 sprint, polish)
- 0.1.0: **첫 배포 가능 stable 릴리즈** — UT/외부 공유 가능
- 0.2.0+: 새로운 sprint 묶음 (백엔드 연동, AI 분석 등)
- 1.0.0: production-ready (백엔드 + auth + 앱 패키징 완료)

### 캐시 헤더 전략

- `/assets/*` (Vite 해시 파일 — `index-CSl2NRQI.js` 등) → 1년 immutable. 코드 바뀌면 hash 바뀌어 새 URL → 안전.
- `/logo/*` (`symbol.png`, `wordmark.png`) → 1일. 로고 변경 시 사용자 캐시가 24시간 내 갱신.
- `/index.html`은 명시 안 함 → Vercel default (no cache, must-revalidate). 새 배포 즉시 반영.

## 7. 요약

ShallWe 프로토타입의 첫 "외부 공유 가능" 버전입니다. 지금까지 7번의 작은 작업(sprint + polish)을 모은 결과물이 안정화돼서, 이제 누구든 인터넷에서 접속할 수 있는 URL을 만들 수 있는 상태가 됐습니다.

이번에 추가된 것:

1. **vercel.json** — Vercel 배포 시 필요한 설정 파일. 핵심은 "deep link 깨짐 방지" — 사용자가 `https://shall-we.vercel.app/home`을 직접 주소창에 치거나 새로고침해도 정상 동작하게 만듭니다.
2. **정적 파일 캐시 설정** — 한 번 다운받은 JS/CSS/이미지를 사용자 브라우저가 다시 다운받지 않게 해서 두 번째 진입부터 빠르게 동작.
3. **버전 0.1.0** — 그동안 0.0.x로 마이너 패치 7번 쌓아왔는데, 이제 안정 단계라 minor 버전 올림. semver 관점에선 "0.1.0 = 첫 배포 가능 릴리즈, 아직 1.0은 아님" 의미.
4. **README 갱신** — 진행 현황에 "Vercel 배포 준비" 추가 + 배포 준비 완료 배지.

이 PR이 main에 머지되면 → GitHub의 inoaole/shall-we에 vercel.json이 올라가고 → Vercel 대시보드에서 "Deploy" 클릭 한 번으로 배포 완료. 받은 URL을 발표·UT·교수님께 공유 가능.

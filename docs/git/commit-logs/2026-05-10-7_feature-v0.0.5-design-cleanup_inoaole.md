## 4. 변경 배경 및 결과

### 배경 (Why)

5번째 PR 직후 사용자 요청 ("이제 마지막으로 전반적으로 디자인이 통일한지, figma랑 다른게 없는지 확인하자"). 서브에이전트로 design system spec (design.md) + tokens.ts + Tailwind config + 12 UI primitive + 4 layout + 16 화면 전수 audit. **26개 finding** (Critical 2 / High 4 / Medium 4 / 그 외 16개) 발견. 사용자가 "C 진행해" — 풀 정리 선택.

핵심 발견:
- `text-red-500`이 토큰 시스템에서 누락됨 (Tailwind colors REPLACE 모드 — error state 색상이 사실상 fallback이거나 누락) → **진짜 버그**
- TodayCard에서 rightSlot으로 inner button 받으면 카드 outer button 안에 button 중첩 → **무효 HTML, 스크린리더 깨짐**
- BackHeader 패턴이 8 화면에 동일 코드로 copy-paste — Header.tsx에 이미 showBack 옵션 있는데 미사용
- Empty state 카드 패턴이 3 곳에 중복 — 추출 안 함
- PoleChoice의 `aria-pressed`는 토글 버튼 의미 → 단일 select group은 `role="radio"`가 정확

### 결과 (What)

- 진짜 버그 2건 수정 (red-500 / 중첩 button)
- inline 패턴 11곳 정리 → 신규 2 컴포넌트 (BackHeader, EmptyState)로 추출
- 코드 LOC -155 (삭제) +185 (추가) = net +30 라인. 신규 컴포넌트 2개 분량을 빼면 실 코드는 감소
- a11y 3건 보강 (PoleChoice radio role, Calendar 한글 aria, 중첩 button 제거)
- design.md 토큰 정확성 강화 (danger 추가 + Sub Title 16 spec 준수)

## 5. 중요 변경 사항

### tokens.ts danger 컬러

design.md §9 미정의 항목으로 "Error / Warning 컬러 — `#E5484D` 권장"이 적혀 있었음. 이번 cycle에 적용 — Input + Textarea의 error state가 정상 동작.

### TodayCard 중첩 button 해결

기존:
```tsx
<button onClick={onClick}>  {/* TodayCard outer */}
  <div>...</div>
  {rightSlot}  {/* Home에서 <button>인증하기</button> 전달 */}
</button>
```

수정:
```tsx
{rightSlot ? (
  // outer: div, body만 button, rightSlot은 sibling
  <div className="...">
    <button onClick={onClick}>{body}</button>
    {rightSlot}
  </div>
) : (
  // 단순 클릭 카드: outer button
  <button onClick={onClick} className="...">{body}</button>
)}
```

Home 인증하기 버튼이 안전하게 sibling으로 배치됨. 무효 HTML 해소.

### BackHeader 추출

기존 8 화면 모두 같은 코드:
```tsx
<header className="px-5 pt-4 pb-3 flex items-center">
  <button onClick={() => navigate(-1)}
    className="text-2xl text-ink leading-none -ml-1 px-1 active:scale-90 transition-transform"
    aria-label="뒤로가기">←</button>
  <h1 className="ml-2 text-title-20 text-ink">화면명</h1>
</header>
```

이제 한 줄:
```tsx
<BackHeader title="화면명" sticky />
```

또는 step navigation override:
```tsx
<BackHeader onBack={handlePrev} rightSlot={<StepPagination .../>} />
```

ChevronLeft (lucide) 아이콘으로 환원해 raw "←" 글자보다 시각적 일관성 ↑.

### EmptyState 추출

3 곳 동일:
```tsx
<div className="bg-white rounded-xl shadow-md p-8 text-center">
  <p className="text-body-14 text-gray mb-4">...</p>
  <button className="text-subtitle-16 text-primary">...</button>
</div>
```

신규 컴포넌트:
```tsx
<EmptyState icon={<Plus size={28} />} message="..." action={{ label, onClick }} />
```

icon prop 추가로 Home 빈 상태에 Plus 아이콘 표시 — 시각 weight 강화.

### PoleChoice a11y

기존: `<button aria-pressed={selected}>` — 토글 버튼 의미 (select all toggle, mute toggle 등에 적합).
수정: `<button role="radio" aria-checked={selected}>` + parent `<div role="radiogroup">` — 단일 select group의 정확한 시맨틱.

스크린리더 사용자가 "선택지 1 of 2, 선택됨" 처럼 정확히 인지 가능.

### Calendar 한글 aria-label

기존: `aria-label="15일 done"` (영어 토큰 leak).
수정: `aria-label="15일 완수, 오늘"` — 한국어 사용자 스크린리더 인지 자연스러워짐.

## 7. 요약

이번 작업은 "기능 추가"가 아니라 "이미 만든 것을 더 일관성 있게" 다듬는 청소 작업입니다. 사용자가 보기엔 큰 차이 없지만 내부적으로 다음이 정리됐습니다:

1. **에러 색상 진짜 버그 수정** — 비밀번호 검증 실패 같은 에러 메시지의 빨간색이 사실 코드에 정의되지 않은 색상이었음 (다른 색이 나오거나 안 나왔을 수 있음). design.md에 미리 적어둔 권장 빨강 (#E5484D) 추가해서 정상 동작.
2. **인증하기 버튼 a11y 버그 수정** — 홈 화면의 챌린지 카드 안에 인증하기 버튼이 들어가 있어서 스크린리더 사용자에게 깨져 보였음. 카드 본체와 인증하기 버튼을 별도 영역으로 분리.
3. **헤더 통일** — 회원가입, 검사, 챌린지 인증, 게시물 상세, 챌린지 만들기, 설정 등 6개 화면에서 "← 화면 제목" 헤더가 각자 다른 스타일로 작성되던 것을 단일 컴포넌트로 통일.
4. **빈 상태 통일** — 챌린지가 없을 때, 게시글이 없을 때 등 3 곳에서 비슷한 카드를 따로 만들었는데 단일 컴포넌트로 통일.
5. **설정 화면 정돈** — 섹션 헤더("알림", "앱", "계정") 글자를 살짝 보강 + 그림자 제거로 시각적 노이즈 감소.
6. **스크린리더 사용자 경험 개선** — 캘린더 셀의 음성 안내가 영어로 나오던 것을 한글로 ("완수", "미래", "오늘") + 양극 2지선다 선택지가 정확한 시맨틱으로 인지되도록.

자동 테스트는 그대로 19개 모두 통과합니다 (PoleChoice의 시맨틱 변경에 맞춰 테스트 selector도 함께 갱신).

코드량은 신규 2 컴포넌트 추가에도 불구하고 inline 패턴 11곳 정리 효과로 실 코드는 줄었습니다 (-90 라인). 향후 비슷한 화면을 새로 만들 때 같은 패턴을 또 손으로 적을 일이 없습니다.

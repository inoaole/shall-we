## 4. 변경 배경 및 결과

### 배경 (Why)

사용자 피드백: "이제 my 화면이 조금 다른 거 같아, 보강해줘"

직전 v0.0.3 polish 작업 후 다른 화면들 (Result / Quiz / Recommend) 톤 일치도가 올라간 반면 MyPage는:
1. 닉네임이 bare `<h1>`로만 노출 — 프로필 페이지의 "정체성" 부족
2. "앱"이라는 어색한 라벨 (Figma 완성본 미존재 — wireframe 작성 시 임의 추가)
3. 설정 row가 흰 카드 + shadow-md로 다른 카드와 시각 구분 없음
4. 누적 활동 (총 인증 횟수, 진행 중 챌린지 수) 표시 없음 → "왜 마이페이지를 와야 하는가" 약함

### 결과 (What)

마이페이지 위계가 4단계로 명확해짐:

1. **프로필 카드** — User 아이콘 (bg-green-tint 원) + 닉네임님 (Title 20) + "ShallWe와 함께 성장 중" 부제목 + 통계 grid (총 인증 / 진행 중 챌린지, 큰 숫자 primary 컬러)
2. **오늘의 챌린지** — TodayCard (있을 때만)
3. **설정** — outlined row (border-only, shadow 없음, 다른 카드와 톤 분리)
4. **내 게시글** — PostFeed (Home과 공유)

진입 시 첫 시선이 "내가 누구인지 + 얼마나 활동했는지"로 가게 됨. 이전엔 nickname만 보고 바로 챌린지로 시선 분산.

## 5. 중요 변경 사항

### Profile card

```tsx
<section className="bg-white rounded-xl border border-gray/15 shadow-sm p-5">
  <div className="flex items-center gap-3 mb-4">
    <div className="w-12 h-12 rounded-full bg-bg-green-tint flex items-center justify-center">
      <User size={22} className="text-primary" />
    </div>
    <div className="flex-1">
      <h1 className="text-title-20 text-ink">{nickname}님</h1>
      <p className="text-body-12 text-gray mt-0.5">ShallWe와 함께 성장 중</p>
    </div>
  </div>
  {/* Stats grid: 총 인증 / 진행 중 챌린지 */}
</section>
```

User 아이콘은 lucide-react. Figma에 avatar 이미지가 없으므로 아이콘 placeholder + bg-green-tint로 ShallWe 컬러 정체성 부여. 후속에 실 사용자 아바타 업로드 기능 추가 시 이 위치에 이미지로 교체.

### Stats grid

```tsx
<div className="grid grid-cols-2 gap-3 pt-4 border-t border-gray/10">
  <div className="text-center">
    <p className="text-title-20 text-primary">{totalCerts}</p>
    <p className="text-body-12 text-gray mt-1">총 인증</p>
  </div>
  <div className="text-center">
    <p className="text-title-20 text-primary">{activeChallengeCount}</p>
    <p className="text-body-12 text-gray mt-1">진행 중 챌린지</p>
  </div>
</div>
```

`state.posts.length` + `state.challenges.length` 단순 집계. 새로고침해도 sessionStorage hydrate로 정확.

### 설정 row 톤 다운

이전: `bg-white shadow-md rounded-xl p-4` — 다른 카드와 동일 시각 weight.
이후: `bg-white border border-gray/15 rounded-xl p-4` (shadow 제거).

근거: 설정은 보조 진입점 — 프로필·챌린지·게시글 보다 시각 weight 낮아야 함. shadow 제거로 위계 분리.

### "앱" 라벨 제거

이전 wireframe.md에 "앱" 섹션 라벨이 있었는데 이건 wireframe 작성 시 의미 분류 위해 적은 메타 라벨. Figma 완성본엔 그런 라벨 없음. 제거하고 "오늘의 챌린지"로 더 직접적인 라벨 사용.

## 7. 요약

마이페이지를 "닉네임만 표시되는 단순 페이지"에서 "사용자 정체성 페이지"로 격상시켰습니다.

이번에 추가된 것:

1. **프로필 카드** — 화면 맨 위에 큰 카드로 사용자 닉네임과 함께 "ShallWe와 함께 성장 중"이라는 부제목, 그리고 사용자 아이콘이 함께 표시됩니다.
2. **활동 통계** — 프로필 카드 안에 "총 인증 N개 / 진행 중 챌린지 N개" 통계가 큰 숫자로 표시됩니다. 마이페이지에 들어오면 본인이 얼마나 활동했는지 한눈에 보이게 했습니다.
3. **설정 위치 정돈** — 설정 버튼을 다른 카드와 시각적으로 구분 (테두리만 있는 형태)해서 부가 메뉴라는 느낌을 줍니다.
4. **"앱"이라는 어색한 라벨 제거** — 그 자리에 "오늘의 챌린지"라는 직접적인 라벨로 교체.

버전은 0.0.4로 올렸고 (Splash와 설정 화면에도 반영), 자동 테스트는 그대로 19개 통과합니다.

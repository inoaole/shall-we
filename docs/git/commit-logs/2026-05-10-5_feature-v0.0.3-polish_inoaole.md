## 4. 변경 배경 및 결과

### 배경 (Why)

S1~S3까지 진행하면서 알림은 `console.warn` stub만 사용 (S2 sprint plan에 "S4에서 sonner 통합"이라 적어둠). 사용자가 사진 picker cancel하거나 챌린지 추가했을 때 시각 피드백이 없어 "정말 됐나?" 의문이 남는 상태. UT 가능 수준으로 만들려면 실제 토스트 필수.

탭 전환도 instant cut만으로는 인터랙션이 딱딱한 인상. 짧은 fade-in 정도의 motion만 추가해도 "살아있는 앱" 느낌.

이전 polish 라운드(v0.0.2 design polish)는 "차분하게 만들기"였다면, 이번 v0.0.3은 "살아있게 만들기" 방향.

### 결과 (What)

- 챌린지 추가, 인증 완료, 데모 가입 등 핵심 액션마다 화면 상단에 sonner 토스트 노출 (2.4초 자동 사라짐, richColors)
- 4탭(다이어리/홈/인증/마이) 전환 시 새 페이지가 살짝 위로 떠오르며 fade-in (220ms)
- 설정 화면의 미구현 메뉴(알림 설정 / 이용약관 / 개인정보 처리방침) 탭 시 alert() 대신 토스트로 "준비 중" 안내
- iPhone SE 등 좁은 화면(≤375pt) 대응을 위한 `xs` Tailwind breakpoint 추가 (실제 적용은 후속 polish에서)
- README 진행 현황 갱신 (S1~S4 모두 완료 표시 + 19 tests pass 배지)

### Bundle size 영향

main bundle 71KB → 119KB gzip (+48KB). sonner (~5KB gzip) + framer-motion 전체 활성화 (~40KB gzip). 프로토타입 단계엔 충분히 acceptable.

## 5. 중요 변경 사항

### sonner 채택 이유

직접 토스트 시스템 구현 vs sonner:
- 직접 구현: ~80줄 + queue 관리 + animation (시간 소요)
- sonner: 1 dependency + `<Toaster />` + `toast.success(...)` 한 줄

prototype 단계엔 명확히 sonner 우위. 추후 디자인 시스템 토큰(green / yellow / red)과 일치하는 richColors 옵션도 매칭.

### Page wrapper 설계

framer-motion + React Router v6의 AnimatePresence는 `<Outlet/>` exit animation이 까다로움. 시도했다가 깜빡임 발생 → exit 포기, **entrance만** 적용.

```tsx
<motion.div
  initial={{ opacity: 0, y: 6 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.22, ease: 'easeOut' }}
>
```

TabLayout에 `<Page key={pathname}>`로 적용 → 탭 전환마다 unmount/mount → fade-in 트리거. 다른 라우트(Quiz, Signup 등)는 단발 진입이라 적용 보류.

### Settings 토스트 통합

`alert()`는 OS 모달 — 디자인 일관성 깨고, UX 유리됨. 모두 notify.error()로 교체 → sonner로 자연스럽게 노출. "준비 중" 메시지 일관성도 확보.

### xs breakpoint 추가만 (실 적용은 후속)

Tailwind config에 `xs: { max: '375px' }` (iPhone SE 폭 이하) 추가. `xs:px-3 px-5` 같은 식으로 좁은 화면에서 padding 줄이기 가능. 실제 모든 화면에 적용은 후속 polish 단계 (사용자가 "iPhone SE에서 깨진다"고 신고할 때 대응).
